# SpectraBreast — HSI + 3D Mesh Pipeline

Registration of hyperspectral images (HSI) onto 3D meshes and generation of spectral point clouds.

---

## What it does

1. Loads the HSI cube (ENVI format) and the 3D mesh
2. Detects ArUco markers on the HSI image and matches them to known 3D positions (JSON)
3. Computes an HSI → render homography to align the two spaces
4. For every mesh pixel with valid 3D coordinates, reads the full spectrum from the HSI
5. Exports the spectral point cloud as `.ply` (CloudCompare), `.npz` (Python), `.csv` (optional)
6. Saves an Excel report with 2D and 3D registration errors


## Installation

### 1. Clone the repository

```bash
git clone <repo-url>
cd SpectraBreast
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

source .venv/bin/activate
```

```bash
pip install -r requirements.txt
```

> **Accelerated ray casting (optional but recommended)**
> If available, install `pyembree` to reduce render time from minutes to seconds.

## Configuration

Open `config.yaml` and edit the parameters. Paths are **relative to the project root**.

```yaml
paths:
  hsi_hdr:    "data/input/SB019_raw.hdr"
  mesh:       "data/input/surface_mesh.ply"
  aruco_json: "data/input/aruco_markers_3d.json"
  output_dir: "data/output"

sample_name: "SB019"

render:
  resolution_mm_per_px: 2.0   # increase for speed, decrease for more detail
```

### Key parameter: `resolution_mm_per_px`

| Value | Speed | Point cloud density | Recommended use |
|-------|-------|---------------------|-----------------|
| `5.0`  | ⚡ fast | low | quick test |
| `2.0`  | ✅ good | medium | standard use |
| `1.0`  | 🐢 slow | high | detailed analysis |
| `0.5`  | 🐢🐢 very slow | very high | GPU NEEDED |

## Usage

### Standard run

```bash
python main.py
```

### Command-line options

```bash
# Use an alternative config (e.g. for a different sample) -> refereed to SB020, sample nominated like that
python main.py --config config_SB020.yaml

# Override the sample name without editing the config
python main.py --sample SB020

# Override the render resolution
python main.py --resolution 1.0

# Validate config and inputs without running (useful for a quick check)
python main.py --dry-run
```

### Example: adding a new sample

1. Copy `config.yaml` to `config_SB020.yaml`
2. Edit the paths and `sample_name: "SB020"` in the new file
3. Run: `python main.py --config config_SB020.yaml`

## Required inputs

| File | Format | Notes |
|------|--------|-------|
| HSI cube | ENVI (`.hdr` + binary data) | BSQ / BIL / BIP interleave |
| 3D mesh | `.ply` or `.obj` | coordinates in **metres** |
| ArUco JSON | `.json` | generated by the 3D reconstruction software |

### ArUco JSON format

```json
{
  "markers": {
    "0": { "corners_3d": [[x,y,z], [x,y,z], [x,y,z], [x,y,z]] },
    "1": { "corners_3d": [[x,y,z], [x,y,z], [x,y,z], [x,y,z]] }
  }
}
```

Coordinates are in **metres** (the pipeline converts them to mm automatically).

## Generated outputs

| File | Description |
|------|-------------|
| `*_render_topview.png` | Orthographic grayscale top-view (depth-shaded) |
| `*_render_turbo.png` | Orthographic top-view coloured with the TURBO colormap |
| `*_registration.png` | HSI + render overlay with ArUco markers and projected points |
| `*_coverage.png` | Coverage map: which HSI pixels are mapped onto the 3D surface |
| `*_registration_errors.xlsx` | 2D (px, mm) and 3D (mm) errors for every ArUco corner |
| `spectral_pointcloud_*.ply` | Spectral point cloud — open in CloudCompare |
| `spectral_pointcloud_*.npz` | Spectral point cloud — fast reload in Python |

### Reloading the point cloud in Python

```python
import numpy as np

data = np.load("data/output/spectral_pointcloud_SB019.npz")
xyz      = data['xyz']         # (N, 3)  coordinates in mm
spectra  = data['spectra']     # (N, bands)  spectral values
wl       = data['wavelengths'] # (bands,)  wavelengths in nm

print(f"Points: {xyz.shape[0]:,}  —  Bands: {spectra.shape[1]}")
```

## Suspicious pixels (optional)

To localise in 3D the regions of interest identified on the HSI (e.g. the output of a segmentation model), edit the `suspicious pixels` section in `main.py`:

```python
mask = np.load("data/input/segmentation_mask.npy")  # binary mask (H, W)
suspicious_pixels, _ = extract_suspicious_centroids(mask, min_area=5)
```

The resulting 3D coordinates are printed to the terminal and visualised on the registration image.

## Troubleshooting

**The mesh is not rendered (0 hits)**
Make sure the mesh units are in metres. If they are already in mm, set `scale_m_to_mm: false` in the `load_mesh` call, or adjust the ArUco JSON accordingly.

**No ArUco markers found on the HSI**
Try changing `hsi_extraction_method` from `mean` to `visible_band` in the config. If the markers are not visible in any band, verify that they are physically present in the image.

**The render is very slow**
Increase `resolution_mm_per_px` (e.g. from `2.0` to `5.0`) or install `pyembree`.

**Error "Data file not found"**
The HSI binary data file must be in the same folder as the `.hdr` and share the same base name (e.g. `SB019_raw.hdr` → `SB019_raw.raw` or `SB019_raw`).
